01 · context

The problem this project solves

Marketing agencies, consultants and growth teams constantly need data on local businesses — to prospect clients, map competitors or audit digital presence. The manual process is slow, repetitive and does not scale.

✗

Without MapLead

Manual Google Maps research, copy-pasting data one by one, disorganized spreadsheets. Hours to map 20 businesses — no standardization, no history.

✓

With MapLead

A single command collects hundreds of businesses with name, rating, reviews, phone, website and hours — all saved to the database automatically, ready for analysis.

To validate the project, I chose the segment of ophthalmologists in Campinas, Brazil — a niche with good data density and high variation in digital presence. The result: 57 businesses and 994 reviews collected in a single run.

Triggering the pipeline

The collection is started with a simple HTTP call — from the terminal, n8n, or any HTTP client.

PowerShell · chamada de teste 
# triggers the scraper directly on the container
Invoke-RestMethod -Method POST -Uri "http://localhost:8000/scrape" `
  -ContentType "application/json" `
  -Body '{"query": "ophthalmologists campinas", "max_results": 20}'

# resposta
{
  "search_id":    1,
  "query":        "ophthalmologists campinas",
  "total_found":  20,
  "message":      "Collection complete. 20 businesses saved (search_id=1)."
}

Joining the tables

SQL · cross-analysis of businesses and reviews 
SELECT
  b.name,
  b.rating         AS overall_rating,
  b.presence_score,
  COUNT(r.review_id) AS total_reviews,
  AVG(r.rating)      AS avg_review_rating
FROM  gmaps_businesses b
JOIN  gmaps_reviews    r ON r.business_id = b.business_id
WHERE b.has_website = false  -- hot leads: no digital presence
GROUP BY b.business_id
ORDER BY avg_review_rating DESC;
